Scope

Explain to a non-tech friend what 'scope' is, and how it works in JavaScript

Scope determines whether a variable is available or unavailable depending on where in the program it is initialised (created), and also depending on where it is (attempting to be) referenced.

Global variables are those that are accessible from the entire program. So it can be acted upon or referenced inside a function that's inside another function.

Local variables live shorter lives. They only exist within the function that they are initialised, and any functions also within that 'home' function. Outside of the 'home' function where it was created, the local variable is unaccessible.

Think of it like a set of boxes. A large one, a medium one and a tiny one. All inside each other like russian dolls. The tiny one inside the medium one that's inside the large.

  • A variable created in the large box can be seen through tiny little view holes in both the smaller boxes, so it is available for all.
  • A variable created in the medium box cannot be seen by the large box, but can be seen by both the medium and tiny boxes.
  • A variable created in the tiny box can only be seen in the tiny box.
